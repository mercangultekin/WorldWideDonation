Use Case : EditProject
Participating Actors : Initiated by User
                                      Aprroved by Admin
Prerequisite Condition : User should initiate DisplayProjectInformation as project leader.
                         
       
Flow of Events : 
 
1) User selects one of his/her started project.
    2) WWDS directs User to project page with editing option for each fields in the "DisplayProjectInformation".
     WWDS reload the page. Make "Project Profile pPictures", "Project Description","Items Needed for The Project"," Project Pictures"
     become editable. "Save Edit" option become available.
5) She edits his/her project. Then he/she clicks on "Save Project."
    6) System send notification message to admin
7) Admin clicks on "show notfication" to see modification has been done to the project. 
    8) System displays the modified project page to admin
9) Admin clicks "Approve" button
    10) WWDS reload the page with the new information for the project.
11) Admin reuploads the project to system which is edited.
Entry Condition : User must be "Logged-In" to the WWDS system  
Exit Condition :  VerifyEdit use case should be initiated by the admin.
